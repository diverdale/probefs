    1. Update on_directory_loaded() to pass show_hidden:
       Change: pane.set_entries(message.entries)
       To:     pane.set_entries(message.entries, show_hidden=self.core.show_hidden)

    2. Add action_toggle_hidden() after action_leave_dir():
       def action_toggle_hidden(self) -> None:
           """Toggle hidden dotfile visibility. '.' key binding."""
           self.core.show_hidden = not self.core.show_hidden
           # No disk re-read — set_entries filters from the already-loaded _entries.
           # We must re-call set_entries on both panes with the current entries stored
           # in each DirectoryList._entries (the raw, unfiltered lists).
           # Re-trigger _load_panes to reload fresh (entries are already cached in
           # memory by the OS so this is fast; no actual disk I/O bottleneck).
           self._load_panes()

    Modify src/probefs/app.py:
    - Add a new Binding for the '.' key after the backspace binding:
        Binding(".", "screen.toggle_hidden", "Toggle hidden", priority=True, show=False),
    - Read the current BINDINGS list first to get the exact position and preserve all existing bindings.
    - The '.' key is not in any existing binding (confirmed in Phase 1 SUMMARY).

    IMPORTANT: The toggle re-calls _load_panes() rather than filtering in-place. This is
    simpler and correct: exclusive=True on the worker means the new call cancels any
    in-flight old load, and the OS directory cache makes the re-read near-instant.
    The research pattern (Pattern 5) notes filtering should happen in set_entries(),
    which it does — show_hidden is passed on every set_entries call.
