<objective>
TDD implementation of ThemeLoader — the validation gateway that ensures no invalid color strings reach Textual's Theme dataclass.

Purpose: Theme is a plain @dataclass with zero validation. ColorParseError from invalid colors only surfaces at paint time with no file context. ThemeLoader owns all validation: it calls Color.parse() for every color field, collects ALL errors (not fail-fast), and raises ThemeValidationError before constructing Theme. This is the enforcement point for THEME-02 and THEME-04.

Output: src/probefs/theme/ sub-package with loader.py (ThemeLoader, ThemeValidationError) and a full pytest test suite following RED→GREEN→REFACTOR.
</objective>

  <behavior>
    ThemeValidationError behavior:
    - __init__(errors: list[str], path: str = "") stores errors list on self.errors
    - str(exc) includes path and bullet list of all errors
    - Raised ONLY after collecting ALL validation errors (never fail-fast)

    ThemeLoader._validate(data: object) -> list[str]:
    - {} (empty dict) -> ["Missing required field: 'name'", "Missing required field: 'primary'"]
    - "not a dict" (str) -> ["Theme file must be a YAML mapping (key: value pairs)"]
    - {"name": "t", "primary": "#5B8DD9"} -> []   (valid minimal theme)
    - {"name": "t", "primary": "notacolor"} -> ["Invalid color for 'primary': 'notacolor' — ..."]
    - {"name": "t", "primary": "#fff", "error": "badcolor"} -> ["Invalid color for 'error': 'badcolor' — ..."]
    - {"name": "t", "primary": "#fff", "dark": "yes"} -> ["Field 'dark' must be a boolean (true/false), got str: 'yes'"]
    - {"name": "t", "primary": "badcolor", "secondary": "alsobad"} -> two errors (both reported, not just first)
    - All 11 color fields (primary, secondary, warning, error, success, accent, foreground, background, surface, panel, boost) are validated when present

    ThemeLoader.load(path: str | Path) -> Theme:
    - Valid YAML file at path -> Theme object with name matching YAML name field
    - Missing file -> FileNotFoundError (not ThemeValidationError)
    - Malformed YAML -> ThemeValidationError(["YAML parse error: ..."], str(path))
    - Invalid schema -> ThemeValidationError([...errors...], str(path))
    - Valid file -> Theme(...) constructed only after _validate() returns []

    ThemeLoader.load_from_string(content: str, source_label: str = "<string>") -> Theme:
    - Valid YAML string -> Theme object
    - Malformed YAML string -> ThemeValidationError(["YAML parse error: ..."], source_label)
    - Invalid schema -> ThemeValidationError([...errors...], source_label)

    ThemeLoader._build_theme(data: dict) -> Theme:
    - Only called internally after _validate() clears
    - kwargs starts with {"name": data["name"], "primary": data["primary"]}
    - All other COLOR_FIELDS present in data are added to kwargs (skip "primary" duplicate)
    - "dark", "luminosity_spread", "text_alpha" added to kwargs if present in data
    - Returns Theme(**kwargs)
  </behavior>
